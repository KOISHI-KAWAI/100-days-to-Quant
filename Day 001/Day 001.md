### 通过Py框架和模块获取数据

一些Python库中已经封装好了数据获取函数或类，例如yfinance(免费), Tushare(有免费额度，需赞助)等. 我们以yfinance为例(参考文档: [yfinance documentation](https://ranaroussi.github.io/yfinance/)) 对于获取单支股票数据，首先构造Ticker对象.

```python
class yfinance.Ticker(ticker, session=None)
```

随后，调用history函数，其参数用于创建PriceHistory类，其返回股票的OHLC和交易量数据.

```python
Ticker.history(*args, **kwargs) → DataFrame
```

history函数主要关注`period`, `interval`, `start`, `end` 参数.

```python
history(period=None, interval='1d', start=None, end=None, prepost=False, actions=True, auto_adjust=True, back_adjust=False, repair=False, keepna=False, rounding=False, timeout=10, raise_errors=False) → DataFrame
```

### 通过HTTP API获取数据

各大交易所通常会提供对外API, 对于频率低的数据请求，通常使用HTTP协议获取量价数据. 这里我们以Binance API为例(参考文档: [Binance Open API](https://developers.binance.com/docs/zh-CN/binance-spot-api-docs/rest-api)), 对于获取加密货币K线数据，使用如下API.

```http
GET <base URL>/api/v3/klines
```

我们可以使用requests模块来发送HTTP GET请求，其get函数通过构造Request对象发送GET请求.

```python
requests.get(url, params=None, **kwargs) → requests.Response
```

get函数主要关注`method`, `url`, `params` 参数.

```python
class requests.Request(method=None, url=None, headers=None, files=None, data=None, params=None, auth=None, cookies=None, hooks=None, json=None)
```

需要注意的是HTTP响应为JSON格式，将其作为json对象格式化到pandas.DataFrame方便后续操作.

```python
df = pandas.DataFrame(rsp.json(), columns=["open_time",...
```

### 通过WebSocket API获取数据

WebSocket 是一种在客户端和服务器之间建立持久双向通信连接的协议, 相较HTTP, 这种方式更适合长期高频的数据请求，并且支持服务器主动推送数据，具体的应用场景是从交易所实时获取例如当前K线数据. 

这里我们同样以Binance API为例(参考文档: [Binance Open API](https://developers.binance.com/docs/zh-CN/binance-spot-api-docs/web-socket-streams)), 对于获取加密货币实时K线数据，使用如下API.

```http
<base wss>/ws/<symbol>@kline_<interval>
```

我们首先构造WebSockeetApp对象，同时重写对象的on_message函数(除了该函数外，还有例如on_open等函数定义了websocket通信时的行为).

```python
class websocket.WebSocketApp(url: str, header: list[str] | dict[str, str] | Callable[[], list[str] | dict[str, str]] | None = None, on_open: Callable[[WebSocketApp], None] | None = None, on_reconnect: Callable[[WebSocketApp], None] | None = None, on_message: Callable[[WebSocketApp, Any], None] | None = None, on_error: Callable[[WebSocketApp, Any], None] | None = None, on_close: Callable[[WebSocketApp, Any, Any], None] | None = None, on_ping: Callable | None = None, on_pong: Callable | None = None, on_cont_message: Callable | None = None, keep_running: bool = True, get_mask_key: Callable | None = None, cookie: str | None = None, subprotocols: list[str] | None = None, on_data: Callable | None = None, socket: socket | None = None)
```

重写on_message函数，该函数定义了当websocket对象接受数据时的行为，记得服务端返回的仍然是JSON格式数据.

```python
def on_message(wsapp,message):
	data = json.loads(message)
	print(f"[-] Todays' real time price: {data["k"]["c"]}")
```

### 通过数据库中间件获取数据

在真实生产环境中，更可能的是从本地/远程数据库中获取数据，这些数据通常是开源API无法获取的数据亦或者经过分析后的重要数据. 受限于本地环境，各位可以搭建中间件例如Mysql，PostgreSQL等. 

我们这里使用最简单的SQLite3, 该数据库不需要服务端. 各位可以使用src文件夹中db_generator.py脚本生成sqlite文件. 首先，我们连接该数据库. 由于刚才所说，我们并不需要指定服务器地址和端口号，只需要指定`database`参数即文件路径便可.

```python
sqlite3.connect(database, timeout=5.0, detect_types=0, isolation_level='DEFERRED', check_same_thread=True, factory=sqlite3.Connection, cached_statements=128, uri=False, *, autocommit=sqlite3.LEGACY_TRANSACTION_CONTROL)
```

这里不多阐述基础的SQL语言，读取数据库中的所有行，并格式化到pandas.DataFrame方便后续操作

```python
df = pandas.read_sql("SELECT * FROM MSFT",conn)
```

### 通过读取本地文件获取数据

在开发环境中，每次编译时获取数据是一种浪费时间和资源的行为. 解决方式就是在第一次获取后以本地文件的方式保存下来，保存格式通常为xlsx, csv等. pandas模块在数据分析方面封装了许多函数，利用read_xxx函数可以直接从支持的格式将文件格式化为pandas.DataFrame.

```python
df = pandas.read_excel(xlsx_path)
```
