## Day 001 获取量价数据的常见手段

### 通过模块中的封装函数获取量价数据

各大数据厂商会将自家接口封装成Python SDK提供开发，根据开发需求来选择数据源. 国内常用: Tushare, Akshare; 国外常用: Yahoo Finance, Polygon等，介于学习成本，这里使用免费的yfinance, Tushare接口实现数据获取.

- #### Yahoo Finance

  首先声明 `yfinace.Ticker` 对象, ticker参数指定股票代码. 该对象的定义如下, 具体参考 [yfinance](https://ranaroussi.github.io/yfinance/index.html):

  ```python
  class yfinance.Ticker(ticker, session=None)
  ```

  > - **ticker** (*str* *|* *tuple**[**str**,* *str**]*) – Yahoo Finance symbol (e.g. “AAPL”) or a tuple of (symbol, MIC) e.g. (‘OR’,’XPAR’) (MIC = market identifier code)
  > - **session** (*requests.Session**,* *optional*) – Custom requests session.

  将Ticker抽象成一只股票，其封装了属性例如 `info`, `news` 等，以及函数 `get_cashflow()`, `history()` 等. 这里使用history函数获取AAPL股票的日线数据:

  ```python
  ticker = yfinance.Ticker("AAPL")
  df = ticker.history(period="1mo", interval="1d")
  ```

- #### Tushare

  所有API都大同小异，Tushare要求在使用前获取Token，对于这类需要验证使用的接口其跟前者yfinance只是多出了一个绑定Token的步骤，具体参考 [Tushare](https://tushare.pro/document/2): 

  ```python
  tushare.set_token("<Your Token>")
  pro = tushare.pro_api("<Your Token>") #或者直接在初始化时指定凭证
  ```

  以获取平安银行(000001.SZ)的日线数据为例，文档给出接口 `tushare.pro_api.daliy()` 接受参数和先前的yfinance大同小异:

  ```python
  pro = tushare.pro_api("<You Token>")
  df = pro.daily(ts_code='000001.SZ', start_date='20260301', end_date='20260401')
  ```

### 通过HTTP请求获取量价数据

这是最常见的数据获取方式，对于没有独立封装成Python SDK的接口通常都会支持HTTP请求. 在Python中通常使用Requests库来实现HTTP请求，这里我将演示使用 Binance 提供的免费HTTP API来获取BTCUSDT的日线数据, 具体参考 [Binance HTTP](https://developers.binance.com/docs/zh-CN/binance-spot-api-docs/rest-api):

```python
rsp = requests.get(url="https://api.binance.com/api/v3/klines", params={
        "symbol":"BTCUSDT",
        "interval":"1d",
        "limit":30
    })
df = pandas.DataFrame(rsp.json()) #返回数据需要序列化成JSON对象
```

如上演示的是requests的快捷函数，对于更进阶的开发有时需要从底层开始. 对于每个请求对象`requests.Request`, 需要建立在会话上，随后使用`requests.session.send()` 发送请求, 具体参考 [Python Requests](https://requests.readthedocs.io/en/latest/):

```python
req = requests.Request(method="GET", url="https://api.binance.com/api/v3/klines", params={
        "symbol":"BTCUSDT",
        "interval":"1d",
        "limit":30
    })

sess = requests.session()
rsp = sess.send(sess.prepare_request(req))
```

### 通过Websocket获取量价数据

对于高频获取数据，HTTP请求显然不是最优方式. 我们通常使用WSS协议来订阅信息流，相较与HTTP请求最大的不同点即是数据并非我们主动请求，而是由服务器定时或在某条件下自动发送到客户端. 这里我将演示使用 Binance 提供的免费WSS API来获取BTCUSDT的日线数据, 具体参考 [Binance WSS](https://developers.binance.com/docs/zh-CN/binance-spot-api-docs/websocket-api). 而WSS客户端的实现通常使用websocket, websocket-client 库实现.

- #### websocket-client

  对于使用Python websocket-client，首先申明客户端对象`websocket.WebSocketApp` 其要求参数中指定回调函数用于配置客户端的行为

  ```python
  ws = websocket.WebSocketApp(url="wss://stream.binance.com:9443/ws",
                                  on_open=on_open,
                                  on_message=on_message)
  ws.run_forever()	
  ```

  这里我们在连接后订阅流随后在客户端接受到消息时打印K线，具体参考 [Python Websocket-client](https://websocket-client.readthedocs.io/en/latest): 

  ```python
  def on_open(wsapp):
      wsapp.send(json.dumps({
          "method":"SUBSCRIBE",
          "params":["btcusdt@kline_1m"],
          "id":1
      }))
  
      print("[+] Websocket opened")
  
  def on_message(wsapp, message):
      msg = json.loads(message)
      print(f"\x20\x20\x20[-] Received kline: {msg.get("k")["c"]}")
  ```

  websocket-client库同时还封装了类似requests中的快捷函数，但是仅适用于临时连接, 由于使用它时通常搭配with语句使用，这里给出示例:

  ```python
  from contextlib import closing
  from websocket import create_connection
  with closing(create_connection("wss://stream.binance.com:9443/ws/btcusdt@trade")) as conn:
      print(conn.recv())
  ```

- #### websockets

  websockets 库相较于websocket-client 在轻量开发中更具优势，后者更适用于多线程开发，而前者支持协程函数

  > 对于不熟悉异步协程函数的读者，这里做简单解释: 协程函数是基于例如网络请求等待时CPU闲置这类现象的解决方式, 当被async修饰词修饰时, 其不返回原先的返回值而是一个协程对象. async通常搭配await语句使用，被await修饰意味着当CPU等待时它允许CPU挂起原先的函数而去执行其他任务

  这里以websockets文档 [Python Websockets](https://websockets.readthedocs.io/en/stable/) 中提供的例子解释websockets的协程用法:

  ```python
  # 服务端
  import asyncio
  from websockets.asyncio.server import serve
  
  async def echo(websocket):
      async for message in websocket:
          await websocket.send(message)
  
  async def main():
      async with serve(echo, "localhost", 8765) as server:
          await server.serve_forever()
  
  if __name__ == "__main__":
      asyncio.run(main())
  ```

  1. asyncio.run(main())： 启动协程事件循环
  2. async with serve()： 指定该with语句为协程上下文管理，意味着在serve()执行的过程中允许await
  3. await server.serve_forever()： 执行serve_forever(), 若在执行期间CPU闲置则挂起main()函数去执行其他任务
  4. async for message in websocket: await websocket.send(message)：每当有客户端连接时发送消息，发送过程中若CPU闲置则挂起函数去执行其他任务

  ```python
  # 客户端
  import asyncio
  from websockets.asyncio.client import connect
  
  async def hello():
      async with connect("ws://localhost:8765") as websocket:
          await websocket.send("Hello world!")
          message = await websocket.recv()
          print(message)
  
  if __name__ == "__main__":
      asyncio.run(hello())
  ```

  1. asyncio.run(hello())： 启动协程事件循环
  2. async with connect()：指定该with语句为协程上下文管理，意味着在connect()执行的过程中允许await
  3. await websocket.send()：执行send(), 若在执行期间CPU闲置则挂起hello()函数去执行其他任务
  4. await websocket.recv()：执行recv(), 若在执行期间CPU闲置则挂起hello()函数去执行其他任务

  > 两个容易疑惑的点：
  >
  > 1. with前修饰async可以让 服务端初始化/客户端连接 的时候await，如果在初始化/连接 的时候CPU闲置(比如连接的时候要等待服务端响应)，那么就挂起函数，去执行其他任务(并不是往下执行，而是去执行其他函数)
  > 2. 协程并不是多线程，他只是让单线程暂时跳过一些任务

### 通过连接数据库获取量价数据

在企业级开发时，内部数据通常是从内网数据库中获取. 数据库又分为关系型(SQL)和非关系型数据库(NoSQL), 后者较少见读者请自行研究. 这里不需要读者自行搭建中间件，我们使用轻量测试环境Sqlite3来演示. 该类型数据库不依赖软件，可以使用单个文件离线的获取数据. 对于构建表，读者可以使用例如Navicat等数据库管理软件实现，这里我们使用pandas封装的sql处理函数实现(事实上pandas也支持读取):

```python
# 写入数据库
conn = sqlite3.connect("./AAPL.sqlite")
df.to_sql("kline", conn, if_exists="replace", index=False)

# 读取数据库
df = pandas.read_sql("SELECT * FROM kline", conn)
```

完整的sqlite3库的使用需要声明操作指针，具体参考 [Python sqlite3](https://docs.python.org/zh-cn/3/library/sqlite3.html): 

```python
conn = sqlite3.connect("./AAPL.sqlite")
cursor = conn.cursor()

res = cursor.execute("SELECT * FROM kline")
df = pandas.DataFrame(res.fetchall())
```

### 通过读取本地文件获取量价数据

对于以.xlsx, .csv 等存储的本地文件，pandas提供了大量的函数可将文件直接读取为 `pandas.DataFrame`，常用的有如下几种，具体参考 [Python pandas](https://pandas.pydata.org/docs/reference/index.html#api): 

| 文件格式: | csv        | xlsx,xls     | json        | sql                          |
| --------- | ---------- | ------------ | ----------- | ---------------------------- |
| **函数:** | read_csv() | read_excel() | read_json() | read_sql(), read_sql_query() |

在读取写入量价数据的过程，作者本身遇到过如下常见问题:

- pandas.to_excel()等函数不支持将带时区格式的时间写入文件

  ```python
  df['time'] = df['time'].dt.tz_localize(None) # 保留本地时间，仅移除时区
  df['time'] = df['time'].dt.tz_convert('UTC').dt.tz_localize(None) #先转换到某时区，再移除时区
  df['time'] = df['time'].astype(str) # 直接转字符串
  ```
